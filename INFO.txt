Analysis and Comparison of Space Efficiency in SparseIntMatrix Class:
a) For a square matrix of N x N elements with m non-zero elements,
the memory required for this SparseIntMatrix implementation is going to be the
amount of memory per primitive type times the amount of non-zero elements, m.
In MatrixEntry, we have three ints (row, col, data), and two MatrixEntry variables (NextRow, NextCol) which
should total 32 bytes (https://javachallengers.com/memory-allocation-with-java/). Thus the total memory required
would be 32*m bytes. Compared to a standard 2D array implementation which would require more space as the
nonzero elements also require memory.

b) For a square matrix, we'll assume N= 100,000 and m= 1,000,000. The SparseIntMatrix implementation
is much more space-efficient. An N x N array would take up N^2 = 10 billion units times 4 bytes equalling
40 GB (40 billion bytes). In comparison, the linked-list implementation requires 32 bytes per non-zero
element. With m = 1,000,000 this totals 32 MB (32 million bytes) or 0.032 GB. In conclusion, it's clear that for large
structures, especially ones with many zero's, a linked list implementation will be more space efficient.
